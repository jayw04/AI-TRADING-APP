## 1. Finding

> The apparent Stage 4 sizing divergence was generated by a non-convergent capped-weight
> implementation. Under the registered five-name, fully invested, 20%-maximum constraints, the
> intended feasible weighting is exactly equal weight.

### 1.1 The feasible set at N=5 is a single point

The registered construction constraints are `max_names = 5`, fully invested, and a hard per-name cap of
`max_position_pct = 0.20`. Then:

```
Σ wᵢ = 1 over 5 names   ∧   wᵢ ≤ 0.20 ∀i     ⟹     wᵢ = 0.20 ∀i
```

The feasible set is the single point (20%, 20%, 20%, 20%, 20%). **No inverse-vol tilt — of any strength,
under any weighting scheme — is expressible at N=5 under the registered constraints.** A cap-respecting
converged hybrid necessarily collapses to equal weight.

### 1.2 Why the harness nevertheless emitted a tilt

`backtest_momentum_stage3.py:109 weigh(...)` computes `0.5·EW + 0.5·inverse-vol`, then enforces the bounds
by iterative clamp-and-renormalize (`:123-128`):

```python
lo, hi = (0.0, 0.20) if n == 5 else (0.075, 0.15)
for _ in range(8):                                    # iterative clamp-to-bounds + renormalize
    w = {t: min(hi, max(lo, x)) for t, x in w.items()}
    s = sum(w.values()) or 1.0
    w = {t: x / s for t, x in w.items()}
    if all(lo - 1e-9 <= x <= hi + 1e-9 for x in w.values()):
        break
```

At n=5 the cap equals the mean weight (`hi = 0.20 = 1/n`). Any vol dispersion pushes a name above the cap;
the clamp pulls it to 0.20; the renormalize — which must restore Σ=1 from a sum below 1 — pushes it back
above. The loop is a fixed-point iteration whose only fixed point is uniform 0.20, approached but not
reached. **It therefore exits on the iteration counter, never on the convergence break, and returns a
vector that violates its own constraint.**

This is specific to N=5. At n=8 and n=10 the bounds are `(0.075, 0.15)` with means of 0.125 and 0.10, so
`hi > 1/n`, the feasible set has interior, and the hybrid is well-posed. The Stage-3 sizing dimension was
therefore meaningful at N=8/N=10 and **degenerate at N=5** — the configuration that became the winner.

## 2. Revised classification

| seam | census v1.0 classification | **revised classification** |
|---|---|---|
| weights | `MATERIAL` · `PERFORMANCE-EVIDENCE-BREAKING` | **`VALIDATION_IMPLEMENTATION_DEFECT`** — truncated clamp-and-renormalize output violated the registered 20% cap on 100% of five-name sessions |

Production equal weighting is **the mathematically feasible implementation of the registered constraints**,
not a deviation from a validated alternative. The census's factual measurements are unchanged; only the
attribution of the divergence changes — from "production drifted from the validated strategy" to "the
validated implementation emitted infeasible weights, and production implements the constraint set
correctly."

**Performance inheritance remains NOT established.** It is not automatic: the historical effect of removing
the defective residual must be quantified before the hold is cleared (§4).

The Stage-3/Stage-4 winner is restated as **N5 / no-cap / equal weight** — the `hybrid_50_50` label at N=5
denotes no distinct feasible strategy and is retired for this configuration. `hybrid_50_50` remains a valid
distinct arm at N=8 and N=10 and is unaffected by this erratum.

### Activation gate

```
BLOCKER (renamed):  WEIGHTING_VALIDATION_DEFECT_IMPACT_NOT_YET_ADJUDICATED
                    (was: PRODUCTION_SIZING_NOT_COVERED_BY_VALIDATED_PERFORMANCE_EVIDENCE)
```

Activation may be authorized **with equal weighting** if the §4 study shows all of:
1. equal weighting preserves the original acceptance verdict;
2. risk and performance differences are immaterial under the registered thresholds;
3. no winner-selection conclusion changes;
4. the production implementation and the audit driver use the same equal-weight sizing seam;
5. the 20% position limit is respected exactly.

If the verdict changes materially, the equal-weight variant requires separate validation.

### Rejected alternatives (owner, 2026-07-22)

- **Port the truncated loop literally** (and raise `max_position_pct` to accommodate it) — **REJECTED.**
  Reproduces a non-convergent numerical artifact rather than an intentional strategy rule; breaches the
  20% concentration limit on every five-name session; changes a risk limit solely to preserve defective
  historical behavior; makes production depend on an arbitrary eight-iteration cutoff. *A result is not
  suitable for production merely because it appeared in the validated backtest — validation can reveal
  that the implementation being validated was defective.*
- **Clamp without renormalizing** — **REJECTED.** Creates a third strategy that is neither the validated
  implementation nor the production one, with variable cash drag, requiring fresh performance evidence.
  A ≤60 bp deviation is not a clean inheritance path.

`max_position_pct` remains **0.20**, unmodified.

## 3. Seam corrections implemented (branch `fix/momentum-daily-weighting-defect-seams`)

Both are correctness fixes to the *configuration surface* and the *measurement apparatus*. **Neither
changes production sizing behavior**, which was, and remains, equal weight capped at 20%.

**3.1 Configuration ambiguity removed — `strategies_user/templates/momentum_daily.py`.** The template
advertised `weighting ∈ {equal, invvol_hybrid}` while no code path read the parameter; sizing was
hardcoded equal. That is the schema/code drift this repo classes as a defect. Now: `choices` narrowed to
`["equal"]`, and `on_init` **fails closed** with `unsupported weighting=…` on any other value — so a stored
param row predating this adjudication is rejected rather than silently sized equal-weight.

**3.2 Single observable sizing seam.** `_per_name_notional(equity, k)` is now the sole source of position
size, consumed by the order path (`_apply_targets`), by the weight-drift trigger (which previously restated
`1/N` independently), and — via `target_weights(target)`, its unit-equity, gross-scaled view — by the §8
audit driver. `drift_audit_driver.capture_seam` previously recomputed `min(1/k, max_position_pct) · gross`
itself, so it agreed with production **by construction and could not have detected a sizing change**: this
is the blind spot that left the weighting question unmeasured until the 21-year census. It now reads the
live class's seam and **fails closed** if a strategy exposes none.

**3.3 Tests.** `test_momentum_daily.py` — schema offers equal only; unsupported weighting fails closed
(3 cases); per-name notional is equal-weight-capped across k∈{1,2,4,5,10}; no target weight ever breaches
the cap for k∈[1,12); weights are gross-scaled and sum to gross; reported weights agree with what the order
path actually sizes. `test_drift_audit_driver.py` — captured weights follow a perturbed live seam (fails
under the old restating implementation); capture fails closed without a seam; captured weights respect the
cap. Full `tests/strategies` suite: **473 passed**, ruff clean.

## 4. Required next work — focused weighting-defect impact study

Scope: **validated truncated weights vs cap-feasible equal weights**, holding historical selections, gross
exposures, trade dates, costs, and data identical. Reported metrics: return, volatility, Sharpe, maximum
drawdown, Calmar, turnover and cost, worst rolling 1-/3-/12-month differences, and whether the Stage-3
winner or the acceptance verdict changes.

**This is a correction-impact analysis for a defective sizing implementation — not strategy discovery, not
retuning, and not another equivalence review.**

Pre-registration: `PREREG_weighting_defect_impact_study_v1.0.md` (this directory). Materiality thresholds
are proposed there and **require owner ratification before the study is run** — thresholding after seeing
